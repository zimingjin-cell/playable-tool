试玩 HTML 检测修改工具 使用说明

Ray · v1.0.3

这个工具是做什么的

这是一个本地 HTML 工具，用来检测和修改 playable HTML 文件。它可以帮助处理常见的试玩跳转、试玩次数、商店链接、背景、语言和覆盖文案等内容。

工具是单文件运行，不需要安装软件，不需要联网，也不会把文件上传到服务器。

推荐使用环境

推荐使用 Chrome 或 Edge 打开。

不推荐使用微信内置浏览器、企业微信内置浏览器或部分旧版浏览器。

如果打开后页面异常、按钮没反应、生成失败，优先换 Chrome 或 Edge 再试。

快速使用步骤

  1. 双击打开 试玩参数修改工具_v1.0.3.html。
  2. 把需要修改的 playable .html 文件拖入工具页面。
  3. 查看 2. 检测结果，确认工具识别到了哪些可修改项。
  4. 在 3. 选择修改 里勾选需要的修改项。
  5. 点击 生成修改版。
  6. 在下方结果区域点击 下载，保存生成后的 HTML。

如果一次导入多个 HTML，可以点击 全部下载。

功能说明

关闭跳转平台

用于拦截常见跳转行为，例如 window.open、mraid.open、OpenStore、InstallFullGame。通常建议默认勾选。

延长试玩次数

用于把常见的步数、操作次数、完成次数、闲置时间等限制调大，适合录屏、测试、反复操作 playable。

清空商店链接

把常见的 iosLink / androidLink 改成 about:blank。一般不需要勾选，只有遇到特别顽固的跳转时再尝试。

背景：更换背景

支持纯色背景和图片材质背景。

注意：Unity / WebGL playable 的背景有时是在游戏画布内部渲染的，不是普通网页背景，因此背景修改不保证对所有试玩都 100% 生效。

强制替换疑似背景贴图

这是高风险选项，用于尝试替换 Unity 场景里的疑似背景贴图。只有当普通背景替换无效时再勾选，它可能误伤大尺寸图集或其他游戏贴图。

文案与语言：强制语言

用于尝试强制浏览器语言，例如 en、zh-CN、ja 等，适合某些会根据浏览器语言自动切换文案的 playable。

添加覆盖文案

在画面上方叠加一行自定义文字，适合录屏时快速加标题、提示语或测试文案。

隐藏常见标题/文案字段

尝试把常见文案颜色字段透明化，例如 titleColor、TextTitleImageColor 等。不是所有 playable 都有效。

常见问题

打开工具后拖不进去文件

请确认使用的是 Chrome 或 Edge。如果是微信或企业微信里直接打开，建议先保存到本地，再用浏览器打开。

生成后的 playable 背景没有变化

可能是该 playable 的背景在 Unity / WebGL 画布内部，或者背景不是普通网页背景。可以尝试勾选 强制替换疑似背景贴图。

生成后的文件只有背景，没有玩法

说明背景替换方案对这个 playable 太激进。请回到工具里取消高风险选项，例如 强制替换疑似背景贴图，重新生成。

使用建议

  • 第一次处理时，只勾选默认项生成一次。
  • 如果只是防跳转，不要额外勾选背景和贴图替换。
  • 如果要换背景，先用普通背景替换。
  • 普通背景无效时，再尝试贴图替换。
  • 每次修改后都建议自己先打开测试，再发给别人。

版本备注

v1.0.3 主要优化了背景修改的兼容性：增加 Unity 常见外层容器背景兜底，保留 canvas 透明处理，并避免过度修改 WebGL 参数导致玩法层消失。
